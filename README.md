# Knowledge Base

结构化知识沉淀与检索。

## 架构

```
知识库/
├── wiki/           # 话题笔记（主体）
├── index.md        # 全局索引
└── log.md          # 操作日志
```

### 设计原则

**知识集中**。同一话题只维护一个笔记，随时间演进，不散落成碎片。

**知识可追溯**。记录来源，才能验证，才能在需要时回到源头。

**知识可碰撞**。新知识与旧知识对话，才能生长。

**知识需整理**。定期梳理，清除冗余，建立关联。

### 文档格式

AI 友好，不追求对人可读。

- API 速查表（类名、方法签名）
- 最小可用代码示例
- 注意事项（常见错误、版本差异）
- 关联引用
