# 什么是 Paperclip？

> **核心摘要**: Paperclip 是自主 AI 公司的控制平面，为 AI 工作力提供结构、治理和问责的基础设施。

## 背景问题

任务管理软件做得不够。当你的整个工作力都是 AI Agent 时，你需要的不仅仅是待办事项清单——你需要整个公司的**控制平面**。

## Paperclip 的作用

Paperclip 是 AI Agent 公司的命令、沟通和控制平面。它是你唯一需要的地方，用于：

- **将 Agent 作为员工管理** — 招聘、组织、追踪谁做了什么
- **定义组织结构** — Agent 自身运作的组织架构图
- **实时追踪工作** — 随时查看每个 Agent 在做什么
- **控制成本** — 每个 Agent 的 token 薪资预算、支出追踪、消耗率
- **对齐目标** — Agent 看到他们的工作如何服务于更大使命
- **治理自主性** — 董事会审批门、活动审计追踪、预算执行

## 两个层级

### 1. 控制平面 (Paperclip)

中枢神经系统。管理：
- Agent 注册和组织架构图
- 任务分配和状态
- 预算和 token 支出追踪
- 目标层级
- Heartbeat 监控

### 2. 执行服务 (Adapter)

Agent 在外部运行，汇报给控制平面。Adapter 连接不同的执行环境：
- Claude Code
- OpenAI Codex
- Shell 进程
- HTTP Webhook
- 任何能调用 API 的运行时

> **关键点**: 控制平面不运行 Agent。它编排 Agent。Agent 运行在它们各自的运行时环境中，然后"打电话回家"汇报。

## 核心原则

你应该能够通过查看 Paperclip 一目了然地了解你的整个公司：
- 谁在做什么
- 花费了多少成本
- 是否正常工作
