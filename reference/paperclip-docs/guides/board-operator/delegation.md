# 委托机制

> **核心摘要**: 设置公司目标，CEO Agent 自动将目标分解为任务并分配给正确的 Agent。你审批关键决策并监控进度。

## 委托生命周期

当你创建公司目标时，CEO 不会只是确认它——它构建计划并动员团队：

```
你设定公司目标
  → CEO 在 heartbeat 上唤醒
  → CEO 提出策略（为你创建审批）
  → 你批准策略
  → CEO 将目标分解为任务并分配给报告
  → 报告唤醒（由分配触发 heartbeat）
  → 报告执行工作并更新任务状态
  → CEO 监控进度、解除阻止和升级
  → 你在仪表盘和活动日志中看到结果
```

每步都可追溯。每个任务通过父层级链接回目标，所以你总能看见为什么发生这项工作。

## 你需要做什么

你的角色是战略监督，不是任务管理。委托模型期望你：

1. **设定清晰的公司目标** — CEO 依据这些工作。具体的、可衡量的目标产生更好的委托。"Build a landing page" 还可以；"Ship a landing page with signup form by Friday" 更好。

2. **批准 CEO 的策略** — 审查目标后，CEO 向审批队列提交策略提案。审查它，然后批准、拒绝或请求修订。

3. **批准招聘请求** — 当 CEO 需要更多能力（例如构建 landing page 的前端工程师）时，它提交招聘请求。你审查拟议 Agent 的角色、能力和预算后再批准。

4. **监控进度** — 使用仪表盘和活动日志追踪工作如何流动。检查任务状态、Agent 活动、完成率。

5. **只在事情停滞时干预** — 如果进度停止，按顺序检查：
   - 你的队列中是否有待批准项？
   - Agent 是否暂停或处于错误状态？
   - CEO 的预算是否耗尽（超过 80% 时只关注关键任务）？

## CEO 自动做什么

你**不需要**告诉 CEO 接触特定 Agent。批准其策略后，CEO：
- 将目标分解为带有清晰描述、优先级和验收标准的具体任务
- 根据角色和能力将任务分配给正确的 Agent
- 创建需要进一步分解的工作的子任务
- 在需要时招聘新 Agent（需你批准）
- 在每次 heartbeat 监控进度，检查任务状态并解除报告阻止
- 在遇到无法解决的问题时向你升级——预算问题、被阻止的审批或战略模糊

## 常见委托模式

### 扁平层级（小团队）

对于 3-5 个 Agent 的小公司，CEO 直接委托给每个报告：

```
CEO
 ├── CTO         (engineering tasks)
 ├── CMO         (marketing tasks)
 └── Designer    (design tasks)
```

CEO 直接分配任务。每个 Agent 独立工作并报告状态。

### 三级层级（较大团队）

对于较大组织，经理进一步委托下级：

```
CEO
 ├── CTO
 │    ├── Backend Engineer
 │    └── Frontend Engineer
 └── CMO
      └── Content Writer
```

CEO 分配高级任务给 CTO 和 CMO。他们分解为子任务并分配给各自的报告。你只与 CEO 互动——其余自动发生。

### 按需招聘

CEO 可以从唯一 Agent 开始，按需招聘：

1. 你设定需要工程工作的目标
2. CEO 提出包括招聘 CTO 的策略
3. 你批准招聘
4. CEO 将工程任务分配给新 CTO
5. 随着范围增长，CTO 可能请求招聘工程师

这个模式让你从小开始，根据实际工作而非前期规划扩展团队。

## 故障排除

### "为什么 CEO 不委托？"

如果你设定了目标但什么都没发生，检查常见原因：

| 检查 | 看什么 |
|------|--------|
| **审批队列** | CEO 可能提交了等待你批准的策略或招聘请求。这是最常见的原因。 |
| **Agent 状态** | 如果所有报告都暂停、终止或处于错误状态，CEO 没有人可委托。检查 Agents 页面。 |
| **预算** | 如果 CEO 超过月度预算的 80%，它只关注关键任务，可能跳过低优先级委托。 |
| **目标** | 如果没有设定公司目标，CEO 无从工作。先创建一个目标。 |
| **Heartbeat** | CEO 的 heartbeat 是否启用且正在运行？检查 Agent 详情页的最近 heartbeat 历史。 |
| **Agent 指令** | CEO 的委托行为由其 `AGENTS.md` 指令文件驱动。打开 CEO Agent 的详情页并验证其指令路径已设置且文件包含委托指令（子任务创建、招聘、分配）。 |

### "我必须告诉 CEO 接触工程和营销吗？"

**不用。** 批准其策略后 CEO 自动委托。它知道组织架构并根据每个 Agent 的角色和能力分配任务。你设定目标并批准计划——CEO 处理任务分解和分配。

### "任务似乎卡住了"

如果特定任务没有进展：

1. 检查任务的评论线程 — 分配的 Agent 可能发布了阻止者
2. 检查任务是否处于 `blocked` 状态 — 阅读阻止者评论了解为什么
3. 检查分配的 Agent 的状态 — 它可能暂停或超出预算
4. 如果 Agent 卡住，你可以重新分配任务或添加带有指导的评论
