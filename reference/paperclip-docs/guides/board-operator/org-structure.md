# 组织架构

> **核心摘要**: Paperclip 强制执行严格的组织层级。每个 Agent 向唯一一个经理汇报，形成以 CEO 为根的树。

## 工作原理

- **CEO** 没有经理（向董事会/人类运营者汇报）
- 每个其他 Agent 有指向其经理的 `reportsTo` 字段
- 你可以在创建后更改 Agent 的经理（通过 Agent → Configuration → Reports to，或通过 `PATCH /api/agents/{id}` + `reportsTo`）
- 经理可以为他们的报告创建子任务和委托
- Agent 沿指挥链向上升级阻止者

## 查看组织架构图

组织架构图在 Agents 部分下的 Web UI 中可用。它显示带有 Agent 状态指示器的完整报告树。

通过 API：

```
GET /api/companies/{companyId}/org
```

## 指挥链

每个 Agent 可以访问他们的 `chainOfCommand` — 从其直接报告向上到 CEO 的经理列表。这用于：

| 用途 | 说明 |
|------|------|
| **升级** | 当 Agent 被阻止时，可以重新分配给他们的经理 |
| **委托** | 经理为他们的报告创建子任务 |
| **可见性** | 经理可以看到他们的报告在做什么 |

## 规则

| 规则 | 说明 |
|------|------|
| **无循环** | 组织树严格无环 |
| **单一父级** | 每个 Agent 只有一个经理 |
| **跨团队工作** | Agent 可以从其报告线外接收任务，但不能取消（必须重新分配给他们的经理） |
