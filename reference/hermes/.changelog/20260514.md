# Hermes 更新日志 - 2026-05-14

> ~50 commits from `80374d4` to `6122a79`

## 重大更新

### 🚀 Codex App-Server Runtime
- 新增可选的 codex app-server 运行时
- 支持 OpenAI/Codex 模型使用独立的 codex-rs/app-server 子进程
- 通过 `model.openai_runtime: codex_app_server` 配置启用
- 24 个端到端测试

### 🎥 统一视频生成工具
- 全新的 `video_generate` 工具，支持插件化后端
- 支持操作：generate / edit / extend
- 模态：text-to-video、image-to-video、video edit、video extend
- 内置后端：
  - **xai/Grok-Imagine**：完整 generate/edit/extend + image-to-video + 参考图片
  - **fal/Veo 3.1**：t2v + i2v，Kling O3 i2v，Pixverse v6 i2v

### ⛓️ EVM 多链技能
- 新增 EVM multi-chain skill（8 条链，14 条命令）
- 合并了 blockchain/base 到 blockchain/evm

### 🔐 安全修复
- 减少 subprocess 调用中的 `shell=True`
- 修复 gateway follow-up 历史记录保留问题

## 消息平台更新

### Slack
- 支持 `!cmd` 作为线程内斜杠命令的替代前缀
- 解决 Slack 线程内不支持原生斜杠命令的问题
- `!queue`, `!stop`, `!model gpt-5.4` 等可在线程中使用

### 消息网关
- 添加 `lazy_deps.ensure()` 到 slack, matrix, dingtalk, feishu 适配器

## CLI 改进

- TUI 中 URL 可点击 + 悬停高亮（任意终端）
- 终端调整大小时保留启动横幅
- 修复 zsh 补全生成
- `lsp` 添加到 `_BUILTIN_SUBCOMMANDS` 跳过插件发现

## Docker 改进

- 分离 `:latest`（仅发布版）和 `:main`（main HEAD）
- 修复 `.venv` 所有权以便 lazy_deps 安装平台包

## 工具系统

- 缓存 `get_nous_auth_status()` 和 `load_env()` 修复慢速 `hermes tools` 菜单
- `video_gen.provider` 在重新配置工具路径时写入
- `skill_view` 名称冲突时拒绝而非猜测

## 性能优化

- 修复 prompt cache 中的 long-lived prefix layout
- system prompt 现在在会话内是字节静态的
- 统一 `nous` 客户端标签 `hermes-client-v<version>`
