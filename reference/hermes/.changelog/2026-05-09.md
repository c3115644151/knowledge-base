## 核心变更

### 🚀 CLI 冷启动性能优化（~21s → ~2.5s）
三项独立优化叠加：
1. **技能目录 mtime 缓存** — `get_external_skills_dirs()` 增加进程内 mtime 缓存，避免每次调用重新解析完整 config.yaml（~120技能安装时 ~10s 节省）
2. **Feishu 懒加载** — `_check_feishu` 从执行 `import lark_oapi` 改为 `importlib.util.find_spec` 检测（约5.2s 节省）
3. **移除 Nous HTTP 探测** — `_web_requires_env` 不再触发 Nous Portal 实时 HTTP 请求（约800ms 节省）

### ⌨️ CLI 快捷键增强
- 新增 `Shift+Enter` 换行支持（需要终端区分 Enter 与 Shift+Enter）
- 明确 Windows Terminal 用户使用 `Ctrl+Enter` / `Ctrl+J`（Alt+Enter 被全屏切换拦截）
- 文档区分 stock Windows Terminal 与 Windows Terminal Preview 1.25+

### ⏰ Cron 定时任务增强
- **deliver=all**：新增 `deliver` 字段特殊值，向所有已连接平台推送结果
- **context_from**：定时任务可链式引用其他任务的最新输出
- **wakeAgent**：pre-check 脚本可选择跳过 Agent 调用，适合高频轮询

### 🔧 其他重要修复
- **patch-tool**：修复 kimi-k2.x 等强制 required-only 模型的参数描述
- **termux**：加固更新路径，uv bootstrap + env guard
- **ollama**：修复切换提供商时残留凭证问题
- ** Nous 认证**：refresh token 改用 header 传输
- **cron**：清理任务时同步删除输出目录
- **TUI**：优化多轮会话分隔符显示、ticker 时长显示
- **quick setup**：新增终端后端选择步骤
- **Google Workspace**：check-auth 增加 disabled_client 检测与 `--check-live` 选项
- **Slack**：manifest 启用 writable app home DMs
- **Gateway**：重启路径捕获 TimeoutExpired 并输出友好错误信息
- **SearXNG**：修复环境变量配置文档（环境变量名称修正）
