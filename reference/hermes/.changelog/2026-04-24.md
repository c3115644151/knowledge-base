## 主要变更

### 1. TUI Fuzzy Match 路径修复

**PR**: #14820 - `fix(tui): @<name> fuzzy-matches filenames across the repo`

**问题**: 当从子目录启动 hermes 时，`@file:apps/web/src/foo.tsx` 这样的路径无法正确解析，因为 agent 会相对于 `apps/web/` 解析。

**修复**:
- 使用 `git -C root rev-parse --show-toplevel` 获取仓库根目录
- 使用 `git -C top ls-files` 列出文件
- 路径转换：`os.path.relpath(top + p, root)`
- 排除 `../` 开头的路径，保持 picker 作用域在 cwd 范围内

**效果**: 从 `apps/web/` 内运行时，`apps/web/src/foo.tsx` 显示为 `@file:src/foo.tsx`

---

### 2. Voice Handlers 修复

**问题**: Rebase 过程中丢失了 `voice.toggle` 和 `voice.record` handlers 以及 panic hooks

**恢复内容**:
- `voice.toggle` - 支持 `on/off/status/tts` 子命令
- `voice.record` - VAD 驱动的连续录音循环
- `_panic_hook` - Gateway crash 日志记录
- `_thread_panic_hook` - 线程异常处理

**新增功能**:
- `voice.tts` - TTS 回复开关
- Agent 回复自动 TTS 朗读 (`speak_text`)
- 状态信息包含 STT/TTS provider 可用性

---

### 3. 新增 hermes_cli/voice.py

**大小**: 548 行

**功能**:
- 进程级语音录制 + TTS API
- Push-to-talk 模式 (`start_recording` / `stop_and_transcribe`)

- 静音检测 (3 次连续静音自动停止)
- 蜂鸣提示音
