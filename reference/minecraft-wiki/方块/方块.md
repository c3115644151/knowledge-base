## 模组开发核心要点 (Modding Priorities)

在 NeoForge 开发中，方块（Block）是构成世界的基础。方块的行为主要通过硬编码的 `Block` 类及其 `BlockState` 管理，而模型、掉落物和配方等则高度**数据驱动化 (Data-Driven)**。

### 1. 注册表与标识符
- **注册表名称**：`minecraft:block` (在 NeoForge 中通过 `Registries.BLOCK` 或 `DeferredRegister.Blocks` 访问)
- **命名空间**：原版方块的 ResourceLocation 均为 `minecraft:<block_name>`（例如 `minecraft:stone`）。
- **方块物品 (BlockItem)**：方块本身并不会自动出现在物品栏中。必须在 `minecraft:item` 注册表中注册一个对应的 `BlockItem` 才能在物品栏中获取该方块。

### 2. 方块属性与状态 (BlockState)
- **方块属性 (BlockBehaviour.Properties)**：在注册方块时，需要传入 `BlockBehaviour.Properties` 来定义其物理属性。常用属性包括：
  - `destroyTime()` / `explosionResistance()`：破坏时间和爆炸抗性（硬度）。
  - `sound()`：方块放置、破坏和行走时的声音（如 `SoundType.STONE`）。
  - `lightLevel()`：发光亮度（0-15）。
  - `noOcclusion()`：非完整方块（如玻璃、台阶）需要设置，以防止光照和渲染错误。
- **方块状态 (BlockState)**：方块的变种（如朝向、是否含水、是否点燃）由 `Property`（如 `DirectionProperty`, `BooleanProperty`）定义，并存储在 `BlockState` 中。通过重写 `createBlockStateDefinition` 方法注册属性。

### 3. 常见开发与交互场景
- **自定义方块逻辑**：通过继承 `Block` 类并重写对应方法：
  - `useWithoutItem` / `useItemOn`：处理玩家右键点击方块的交互逻辑（1.21+ 更新了交互方法的签名）。
  - `stepOn` / `fallOn`：处理实体走在方块上或掉落在方块上的逻辑（如史莱姆块的弹跳）。
  - `tick` / `randomTick`：处理方块的计划刻和随机刻更新（如农作物生长）。
- **数据驱动配置**：
  - **掉落物**：存放在 `data/<namespace>/loot_table/blocks/` 目录下。使用 JSON 定义掉落逻辑，而非在代码中硬编码。
  - **方块模型与材质**：通过 `assets/<namespace>/blockstates/` (方块状态映射) 和 `assets/<namespace>/models/block/` (模型定义) 绑定材质。
  - **方块标签 (Block Tags)**：存放在 `data/<namespace>/tags/block/` 目录下。用于定义挖掘等级（如 `#minecraft:mineable/pickaxe`）、是否需要特定工具（如 `#minecraft:needs_iron_tool`）等。

### 4. 开发避坑指南
- **BlockEntity (方块实体)**：普通的方块无法存储复杂的动态数据（如容器的物品、熔炉的进度）。如果需要存储数据或每刻执行复杂逻辑，必须绑定 `BlockEntity`，并注册到 `Registries.BLOCK_ENTITY_TYPE`。
- **1.21+ 挖掘工具机制更新**：方块的挖掘工具和挖掘等级现在完全依赖于**标签 (Tags)**，不再通过代码中的 `Properties.requiresCorrectToolForDrops()` 显式指定具体的工具类。必须在数据包中为方块添加对应的工具标签。
- **单例模式**：每个注册的 `Block` 实例在游戏中是单例的。**绝对不要**在 `Block` 类中定义非常量的成员变量来存储与特定坐标相关的状态，必须使用 `BlockState` 或 `BlockEntity` 存储坐标相关的数据。
