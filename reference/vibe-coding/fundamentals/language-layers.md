# 语言层要素 (Language Layers) - L1-L12

> **框架类型**：代码理解层级
> **核心问题**：看不懂代码 = 不懂其中某一层模型
> **AI 执行要点**：逐层理解，精准定位卡点

---

## 一、层级总览

| 层级 | 名称 | 决定你能不能… |
|:---|:---|:---|
| L1 | 控制语法 | 写出能跑的代码 |
| L2 | 内存模型 | 不写出隐式 bug |
| L3 | 类型系统 | 不靠注释理解代码 |
| L4 | 执行模型 | 不被 async/并发坑 |
| L5 | 错误模型 | 不漏资源/不崩 |
| L6 | 元语法 | 看懂"不像代码的代码" |
| L7 | 范式 | 理解不同风格 |
| L8 | 领域/生态 | 看懂真实项目 |
| L9 | 时间模型 | 控制性能与时序 |
| L10 | 资源模型 | 写出高性能系统 |
| L11 | 隐含契约 | 写出可上线代码 |
| L12 | 设计意图 | 成为架构者 |

## 二、基础层（L1-L3）

### L1: 控制语法

```
包含：变量、if/else、for/while、函数/return

触发：只能看懂教学代码
局限：无法理解真实项目
```

### L2: 数据与内存模型

```
包含：值 vs 引用、栈 vs 堆、拷贝 vs 共享、指针/引用、可变/不可变

触发：C/C++/Rust/Python 差距的根源

IF 看不懂引用传递 THEN
  → 需要理解 L2 内存模型
END IF

示例问题：
├─ "a = b" 在 Python 中是引用复制还是值复制？
├─ "int *p = &a" 是什么意思？
└─ 什么是深拷贝和浅拷贝？
```

### L3: 类型系统

```
包含：静态类型/动态类型、类型推导、泛型/模板、类型约束、Null/Option

触发：跨语言理解或类型错误

IF 看不懂泛型约束 THEN
  → 需要理解 L3 类型系统
END IF

示例问题：
├─ "fn foo<T: Copy>(x: T) -> Option<T>" 是什么意思？
├─ TypeScript 的 "as" 断言安全吗？
└─ 为什么 Rust 用 Option 而不是 null？
```

## 三、执行层（L4-L6）

### L4: 执行模型

```
包含：同步 vs 异步、阻塞 vs 非阻塞、线程 vs 协程、事件循环、内存可见性

触发：异步编程或并发问题

IF 不知道 "await fetch()" 什么时候执行 THEN
  → 需要理解 L4 执行模型
END IF

关键问题：
├─ 什么时候执行？
├─ 谁在等谁？
└─ 是否有竞态条件？
```

### L5: 错误模型

```
包含：异常 vs 返回值、panic/throw、RAII、defer/finally

触发：资源泄漏或崩溃问题

IF 不确定 defer 什么时候执行 THEN
  → 需要理解 L5 错误模型
END IF

关键问题：
├─ 失败会发生什么？
├─ 资源是否正确释放？
└─ 错误是否向上传播？
```

### L6: 元语法

```
包含：宏、装饰器、注解、反射、代码生成

触发：遇到"不像代码的代码"

IF 看不懂 "@cache def f()" 做了什么 THEN
  → 需要理解 L6 元语法
END IF

关键问题：
├─ 它在改写什么代码？
├─ 装饰器什么时候执行？
└─ 反射的性能代价是什么？
```

---

## 四、范式层（L7-L8）

### L7: 语言范式

```
包含：OOP、FP、过程式、声明式

触发：理解不同编程风格

IF 看不懂 "map (+1) xs" THEN
  → 需要理解 L7 函数式范式
END IF

关键问题：
├─ 这是对集合做变换，不是循环
├─ 副作用在哪里？
└─ 引用透明性是否满足？
```

### L8: 领域语法

```
包含：SQL、正则、Shell、DSL、框架约定

触发：理解真实项目

IF 不懂 SQL 的 "SELECT * FROM t WHERE id IN (...)" THEN
  → 需要理解 L8 领域语法
END IF

关键问题：
├─ 这是什么领域的 DSL？
├─ 框架约定的隐含规则是什么？
└─ 语法糖如何展开？
```

## 五、架构层（L9-L12）

### L9: 时间维度模型

```
包含：一次性计算 vs 多次复用、延迟执行、重复执行

触发：性能/Bug/竞态/重复执行问题

IF 不确定 "@lru_cache" 是计算一次还是每次执行 THEN
  → 需要理解 L9 时间模型
END IF

关键问题：
├─ 它在什么时候跑？
├─ 它会跑多久？
├─ 它是否重复跑？
└─ 它是否延迟跑？
```

### L10: 资源模型

```
包含：CPU 密集 vs IO 密集 vs 内存绑定 vs 网络阻塞

触发：性能优化和并发模型选择

IF 不考虑数据在哪和处理类型 THEN
  → 需要理解 L10 资源模型
END IF

关键问题：
├─ 数据在哪？（内存/磁盘/网络）
├─ 处理是算还是等？
├─ 能不能并行？
└─ 能不能批量？
```

### L11: 隐含契约

```
包含：是否允许返回 None、是否允许 panic、是否线程安全

触发：生产级代码问题

IF 不确定函数是否允许返回 null THEN
  → 需要理解 L11 隐含契约
END IF

关键问题：
├─ 作者在防什么？
├─ 什么情况会导致崩溃？
└─ 隐含的前置条件是什么？
```

### L12: 设计意图

```
包含：防 bug/防误用/性能换可读性/为未来扩展留钩子

触发：代码审查、架构设计、API 设计

IF 不理解 "Result<U, E>" 为什么比异常更好 THEN
  → 需要理解 L12 设计意图
END IF

关键问题：
├─ 作者为什么这么写？
├─ 这个设计取舍是什么？
└─ 还有更好的方案吗？
```

## 六、工程级自测题

```
当你看到一段陌生代码时，问自己：

1. 我知道它的数据在哪吗？ (L2/L10)
   └─► IF 否 → 需要理解内存/资源模型

2. 我知道它什么时候执行吗？ (L4/L9)
   └─► IF 否 → 需要理解执行/时间模型

3. 我知道失败会发生什么吗？ (L5/L11)
   └─► IF 否 → 需要理解错误/契约模型

4. 我知道作者在防什么吗？ (L12)
   └─► IF 否 → 需要理解设计意图

✅ 全 YES = 真·100% 看懂
```

## 七、卡点定位表

| 卡住表现 | 实际缺失 | 解决方案 |
|:---|:---|:---|
| "这行代码看不懂" | L2/L3 | 内存模型/类型系统 |
| "为啥结果是这样" | L4 | 执行模型 |
| "函数去哪了" | L6 | 元语法 |
| "风格完全不一样" | L7 | 范式 |
| "这不是编程吧" | L8 | 领域语法 |
| "性能怎么这么差" | L9/L10 | 时间/资源模型 |
| "怎么又崩了" | L5/L11 | 错误/契约模型 |
| "为什么要这样写" | L12 | 设计意图 |

## 八、各语言层级对照

| 层级 | Python | Rust | Go | JavaScript |
|:---|:---|:---|:---|:---|
| L2 内存 | 引用为主，GC | 所有权+借用 | 值/指针，GC | 引用为主，GC |
| L3 类型 | 动态，type hints | 静态，强类型 | 静态，简洁 | 动态，TS可选 |
| L4 执行 | asyncio/GIL | tokio/async | goroutine/channel | event loop |
| L5 错误 | try/except | Result/Option | error返回值 | try/catch/Promise |
| L6 元语法 | 装饰器/metaclass | 宏 | go generate | Proxy/Reflect |
