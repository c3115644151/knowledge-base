# 问题求解能力 (Problem Solving Framework)

> **框架类型**：核心能力框架
> **核心公式**：问题求解 = 定义问题 × 构造解法 × 验证结果
> **AI 执行要点**：将所有复杂能力归结为问题求解能力

---

## 一、核心定义

```
问题 = 目标状态 - 当前状态

问题求解 = 定义问题 × 构造解法 × 验证结果
```

---

## 二、问题定义要素

### 2.1 四要素

| 要素 | 描述 | 示例 |
|:---|:---|:---|
| 目标 | 要解决到什么程度 | "可用"还是"最优" |
| 现状 | 现在是什么情况 | 当前代码状态、环境状态 |
| 差距 | 目标和现状之间差了什么 | 功能缺失、性能不足 |
| 标准 | 怎样算解决了 | 可运行的代码、通过测试 |

### 2.2 约束条件

| 类型 | 描述 | 示例 |
|:---|:---|:---|
| 时间 | 完成期限 | 明天之前 |
| 资源 | 可用资源限制 | 只能使用 Python |
| 规则 | 必须遵守的规则 | 不能调用付费 API |
| 风险 | 可接受的风险范围 | 数据不能丢失 |
| 能力 | 能力上限 | 不熟悉 Rust |

### 2.3 分析对象

| 对象类型 | 描述 | 示例 |
|:---|:---|:---|
| 事 | 需要完成的任务 | 实现某个功能 |
| 人 | 涉及的人 | 用户、管理员 |
| 系统 | 涉及的系统 | 数据库、API |
| 信息 | 需要处理的数据 | 用户输入、配置文件 |
| 资源 | 可用的资源 | 库、云服务 |
| 环境 | 运行的环境 | Linux、浏览器 |

## 三、求解路径

### 3.1 拆解

```
复杂问题 → 可独立完成的小单元

IF 问题复杂 THEN
  → 拆解为子问题
  → 确定子问题间的依赖
  → 按依赖顺序解决
END IF
```

### 3.2 排序

```
IF 多个子问题并行 THEN
  → 确定优先级
  → 先解决依赖其他问题的部分
END IF
```

### 3.3 试错

```
IF 第一次尝试失败 THEN
  → 分析失败原因
  → 调整方案
  → 重新尝试
END IF
```

### 3.4 反馈

```
IF 执行后结果不符合预期 THEN
  → 分析偏差
  → 获取更多信息
  → 修正方案
END IF
```

### 3.5 修正

```
IF 方案不可行 THEN
  → 回退到上一个可行状态
  → 尝试其他路径
END IF
```

## 四、问题求解决策树

```
FUNCTION solve_problem(问题描述)
  │
  ├─► 1. 定义问题
  │     ├─► 目标是什么？
  │     ├─► 现状是什么？
  │     ├─► 差距是什么？
  │     └─► 判断标准是什么？
  │
  ├─► 2. 分析约束
  │     ├─► 时间约束？
  │     ├─► 资源约束？
  │     ├─► 技术约束？
  │     └─► 风险约束？
  │
  ├─► 3. 识别对象
  │     ├─► 涉及什么系统？
  │     ├─► 涉及什么数据？
  │     └─► 涉及什么资源？
  │
  ├─► 4. 构造解法
  │     ├─► 拆解为子问题
  │     ├─► 设计解决路径
  │     └─► 评估方案可行性
  │
  ├─► 5. 执行验证
  │     ├─► 分步执行
  │     ├─► 每步验证
  │     └─► 反馈修正
  │
  └─► 6. 确认解决
        ├─► 符合判断标准？
        ├─► 文档更新？
        └─► 总结经验？

END FUNCTION
```

## 五、方法论映射

### 5.1 需求不清

```
→ 现象学还原：现象/意图/情境/边界
→ 概念工程：术语/定义/边界/不变量
```

### 5.2 质量不稳

```
→ 可证伪主义：每个断言配反例测试
→ 形式化方法：类型/schema/不变式
```

### 5.3 排错提效

```
→ 溯因推理：候选原因 + 区分性实验
→ 贝叶斯更新：先验 → 实验 → 后验
```

### 5.4 复杂系统

```
→ 系统论：边界与反馈回路
→ 计算哲学：reference vs optimized
```

## 六、能力压缩

```
问题求解能力 = 准确定义问题，并在目标、约束、对象之下，
               设计并执行有效求解路径的能力

本质：
├─ 所有其他能力是问题求解能力在不同场景的表现
├─ 学习能力 = 解决"如何更快获得有效知识"的问题
├─ 决策能力 = 解决"在不确定条件下如何选更优方案"的问题
├─ 沟通能力 = 解决"如何让信息被准确接收"的问题
├─ 管理能力 = 解决"如何通过资源配置达成目标"的问题
└─ 创新能力 = 解决"旧解法不够用时如何找到新解法"的问题
```

---

## 七、AI 执行要点

```
IF 遇到任何任务 THEN
  → 首先准确定义问题
  → 分析约束条件和对象
  → 设计求解路径
  → 分步执行并验证
  → 必要时修正方案
END IF

IF 问题描述不清晰 THEN
  → 使用现象学还原
  → 明确现象/意图/情境/边界
  → 再继续求解
END IF
```
