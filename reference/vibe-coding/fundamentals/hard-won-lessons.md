# Hard-Won Lessons Protocol

## 适用场景
- 开始新项目或新任务时
- 评估技术方案和工具时
- 避免重复踩坑时

## 触发条件
- 准备动手开发前
- 选择技术栈或开源方案前
- 需要决策优先级时

---

## IF-THEN Rules

### Critical: Research Before Build

**IF** starting a development task  
**THEN** apply: 10% execution, 70% research and discussion

**IF** about to write code  
**THEN** FIRST search for all relevant materials and discuss with AI  
**THEN** align on: What? Why? How? Is this optimal?

**IF** using Perplexity or similar tools  
**THEN** use them to find existing solutions before building

---

### The 70-30 Rule

**BEFORE** writing first line of code:

1. **What is this?**
   - Define the problem clearly
   - Identify the core technology/domain

2. **Why this approach?**
   - Justify the technical choice
   - Consider alternatives

3. **How to implement?**
   - Map out architecture
   - Identify dependencies and constraints

4. **Is this optimal?**
   - Compare with existing solutions
   - Check for better open-source options

---

### Common Failure Pattern

**IF** you skip research and start coding immediately  
**THENEN you will likely discover:
- A better open-source solution already exists
- Significant rework is required
- Time wasted on solved problems
