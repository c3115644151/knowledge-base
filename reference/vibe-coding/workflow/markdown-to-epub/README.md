# Markdown 转 EPUB 工作流

> 基于 Calibre ebook-convert 的可复现转换流程

## 执行协议

### 触发条件

| 场景 | 触发方式 |
|:---|:---|
| 电子书制作 | 提供 Markdown 文件路径，转换为 EPUB 格式 |
| 阅读适配 | 需要在阅读器/手机上阅读 Markdown 内容 |
| 文档归档 | 将 Markdown 转换为可移植电子书格式 |
| 批量转换 | 多个 Markdown 文件批量转为 EPUB |

### 输入输出规范

#### 输入
| 类型 | 说明 | 必需 |
|:---|:---|:---|
| Markdown 路径 | 源文件路径 (绝对/相对) | 是 |
| 元数据 | 标题/作者/语言 | 部分必需 |
| 输出 EPUB 路径 | 目标文件路径 | 否 (默认同目录) |

#### 输出
| 类型 | 说明 | 格式 |
|:---|:---|:---|
| EPUB 文件 | 可阅读的电子书 | `.epub` |
| 证据报告 | 转换日志与自检结论 | `report.json` |
| 转换日志 | 详细命令输出 | `conversion.log` |

### 核心步骤详解

#### Step 1: 定位上下文
```
输入: Markdown 路径 + 工作目录
动作:
  - Test-Path 确认文件存在
  - Get-Content 读取前60行提取标题/作者
  - 注意编码: UTF-8
输出: 
  - 输入文件绝对路径
  - 元数据候选 (标题/作者/语言)
检查点: 文件存在且可读
```

#### Step 2: 自动补全
```
输入: Step1 的元数据候选
动作:
  - ebook-convert --version 记录工具版本
  - 确认编码 (UTF-8)
  - 补全缺失元数据
输出:
  - 工具与路径: ebook-convert (Calibre)
  - 最终元数据
检查点: ebook-convert 可用
```

#### Step 3: 计划
```
输入: Markdown 路径 + 输出路径 + 元数据
动作:
  路径A (最小路径):
    - 直接调用 ebook-convert
  路径B (稳健路径):
    - 使用 build_epub.py 脚本
    - 生成报告与证据
  - 确认覆盖策略
输出: 执行命令 + 预计输出路径
检查点: 命令非交互式，路径含空格时加引号
```

#### Step 4: 执行取证
```
输入: 执行路径 + 输出路径 + 元数据
动作:
  路径A:
    ebook-convert "<input.md>" "<output.epub>" \
      --title "<标题>" \
      --authors "<作者>" \
      --language "<语言>"
  路径B:
    python build_epub.py \
      --input-md "<input.md>" \
      --output-epub "<output.epub>" \
      --title "<标题>" \
      --authors "<作者>" \
      --language "<语言>"
输出:
  - EPUB 文件
  - report.json + conversion.log
检查点: 输出文件存在且 > 10KB
```

#### Step 5: 自检验收
```
输入: 输出 EPUB 路径 + 报告 JSON 路径
动作:
  结构完整性:
    - has_opf = true
    - has_ncx_or_nav = true
  内容验证 (抽样):
    - 标题/作者存在于 HTML
    - 表格转换为 <table>
输出: 验收结论 (通过/不通过)
检查点: 目录存在，元数据命中，关键内容存在
```

#### Step 6: 交付落盘更新
```
输入: 通过验收的 EPUB + 证据材料
动作:
  - 复制到目标目录
  - 更新资产文档 (版本 + 变更记录)
输出: 最终交付物路径列表
检查点: 文件齐全且可打开
```

### 检查点清单

| 检查项 | 标准 | 修复措施 |
|:---|:---|:---|
| 文件存在 | Test-Path = True | 确认路径 |
| 编码正确 | UTF-8 解码成功 | 指定编码参数 |
| 工具可用 | ebook-convert --version 有输出 | 安装 Calibre |
| 输出非空 | 文件大小 > 10KB | 检查错误日志 |
| OPF 存在 | has_opf = true | 调整转换参数 |
| 目录存在 | has_ncx_or_nav = true | 添加 --level1-tiltle 等参数 |
| 内容完整 | 关键文本存在于 EPUB | 验证抽样 |

### 工具要求

| 工具 | 版本 | 说明 |
|:---|:---|:---|
| Calibre ebook-convert | ≥ 5.0 | EPUB 转换核心工具 |
| Python | ≥ 3.8 | 运行构建脚本 (可选) |

### 分支处理

#### E.1 信息不足
```
触发: 无法提取标题/作者/语言
最少问题 (最多3个):
  1. 输出路径与覆盖策略
  2. 书籍元数据 (标题/作者)
  3. 是否需要证据报告
默认策略: 追加日期，语言默认 zh-CN
```

#### E.2 信息冲突
```
触发: 元数据不一致或覆盖策略不明确
决策: 以用户确认的为准，否则以 Markdown 头部为准
需要批准: 覆盖文件 / 清理构建目录
```

#### E.3 时间紧 (MVP)
```
触发: 快速拿到可读 EPUB
MVP:
  - 直接 ebook-convert 生成
  - 最小自检 (能打开 + 目录存在)
后续: 增加报告、内容校验、图片归一化
```

#### E.4 命令失败/环境异常
```
诊断:
  1. ebook-convert --version 是否可用
  2. 记录完整错误输出
  3. 检查输入文件编码与路径
回退:
  - 安装/修复 Calibre
  - 不清理构建目录，换新输出名
```

### 交付物命名

```
命名规则: YYYYMMDD-Markdown转EPUB-<版本>.md
落盘路径: <项目根>/docs/Markdown转EPUB/

证据索引:
  EVID-001: 工具版本 (ebook-convert --version)
  EVID-002: 转换命令 (完整命令行)
  EVID-003: 转换日志 (conversion.log)
  EVID-004: 自检结论 (目录/元数据/内容)
```
