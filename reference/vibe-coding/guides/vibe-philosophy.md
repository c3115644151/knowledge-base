# Vibe Coding 哲学原理

> **核心思想**：道生一，一生二，二生三，三生万物
> **AI 执行要点**：理解 AI-人类协作的哲学基础

---

## 一、核心公式

```
一：安装一个 AI CLI，获得与 AI 对话的能力
二：AI 能读写一切文件，你不再需要手动编辑
三：AI 能配置一切环境，安装依赖、部署项目
万物：AI 生成代码、文档、测试、脚本——一切皆可生成
```

---

## 二、心法

```
我是 AI 的寄生者，没有 AI 我失去一切能力。

人类：描述意图、验证结果、做决策
AI：理解意图、执行操作、生成产出
```

---

## 三、范式转变

```
传统编程：
├─ 人写代码
├─ 人调试代码
└─ 人维护代码

Vibe Coding：
├─ 人描述意图
├─ AI 生成代码
├─ 人验证结果
└─ 人做决策
```

---

## 四、AI 执行规则

### 4.1 角色分工

```
IF 人类提出需求 THEN
  → AI 负责理解、执行、生成产出
  → 人类负责描述意图、验证结果、做决策
END IF

IF AI 生成代码 THEN
  → 必须包含输入输出示例
  → 必须说明可能失败的情况
  → 必须标注需要人类确认的决策点
END IF
```

### 4.2 规划驱动

```
IF 人类要求实现功能 THEN
  → AI 必须先输出实施计划
  → 等待人类确认后再执行
  → 每完成一步报告进度
END IF
```

### 4.3 模块化约束

```
IF 生成新代码 THEN
  → 必须拆分到多个文件
  → 每个文件职责单一
  → 禁止 monolith
END IF
```

## 五、胶水编程扩展

```
传统 Vibe Coding 的痛点：
├─ AI 幻觉
├─ 复杂性爆炸
└─ 门槛过高

胶水编程的解法：
├─ 零幻觉：只使用已验证的成熟代码
├─ 复杂性归零：每个模块都是久经考验的轮子
└─ 门槛消失：你只需要描述"连接方式"
```

---

## 六、AI 能力边界

```
AI 能做：
├─ 生成代码
├─ 读写文件
├─ 执行命令
├─ 配置环境
└─ 生成文档

AI 不能做（需要人类）：
├─ 做最终决策
├─ 验证业务价值
├─ 承担风险
└─ 判断道德/法律合规
```
