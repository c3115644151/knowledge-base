# 网络环境配置协议

> AI 指导用户完成网络代理配置，确保能正常访问 GitHub、Google、Claude 等服务

---

## 前置条件检查清单

```
IF 用户报告无法访问 GitHub/Google/Claude
THEN 执行本配置协议
ELSE 跳过，跳转至 [开发环境搭建](./开发环境搭建.md)

检查项：
□ 操作系统确认（Windows/macOS/Linux/Android）
□ 已有网络代理服务订阅链接
□ 已安装 FlClash 或需要下载
```

---

## 配置步骤

### 步骤 1：下载 FlClash

```
IF 用户操作系统 = Windows
THEN 下载 FlClash-x.x.x-windows-setup.exe
     执行: https://github.com/chen08209/FlClash/releases

IF 用户操作系统 = macOS
THEN 下载 FlClash-x.x.x-macos.dmg

IF 用户操作系统 = Linux
THEN 下载 FlClash-x.x.x-linux-amd64.AppImage

IF 用户操作系统 = Android
THEN 下载 FlClash-x.x.x-android.apk
```

### 步骤 2：导入订阅

```
IF FlClash 已安装 = TRUE
THEN 执行以下操作：
     1. 打开 FlClash
     2. 点击 配置 → 添加
     3. 选择 URL 导入
     4. 粘贴订阅链接
     5. 确认并等待节点加载
ELSE 引导用户先完成下载安装
```

### 步骤 3：开启代理

```
IF 节点已加载 = TRUE
THEN 按顺序设置：
     | 设置项              | 操作          |
     |---------------------|---------------|
     | 虚拟网卡 (TUN)      | 开启 ✓        |
     | 系统代理            | 开启 ✓        |
     | 代理模式            | 全局模式      |

IF 设置成功
THEN 确认 FlClash 主界面显示 "已连接"
```

---

## 验证检查点

```
验证命令：
  curl -I https://www.google.com
  curl -I https://github.com

IF 返回结果包含 "HTTP/2 200"
THEN 配置成功
ELSE 执行故障排查
```

---

## 常见问题处理

| 问题 | 诊断条件 | 解决方案 |
|:-----|:---------|:--------|
| 节点连不上 | 连接超时 | 切换其他节点；检查订阅是否过期 |
| 部分应用不走代理 | 特定应用无法访问 | 确保 TUN 模式（虚拟网卡）已开启 |
| 终端不走代理 | curl 命令失败 | 手动设置环境变量 |
| TUN 模式失败 | 权限错误 | 以管理员/root 权限运行 FlClash |

### 终端手动代理配置

```
IF TUN 模式不可用
THEN 执行：
     export https_proxy=http://127.0.0.1:7890
     export http_proxy=http://127.0.0.1:7890
```
