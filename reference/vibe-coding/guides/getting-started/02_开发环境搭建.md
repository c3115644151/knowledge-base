# 开发环境搭建协议

> AI 指导用户根据操作系统完成开发环境配置

---

## 前置条件检查清单

```
IF 用户已完成网络环境配置
THEN 继续执行
ELSE 引导跳转至 [网络环境配置](./网络环境配置.md)

检查项：
□ 操作系统确认
□ 终端访问权限确认
□ 网络代理已配置（如需要）
```

## 配置步骤

### Windows 用户

#### 方案 A：WSL2 + Linux 环境（推荐）

```
IF 用户选择 WSL2 方案 AND 操作系统 = Windows
THEN 按顺序执行：
     
     STEP 1: 安装 WSL2
       IF PowerShell 可用 = TRUE
       THEN 执行: wsl --install
       ELSE 引导手动安装

     STEP 2: 安装 Ubuntu
       IF WSL2 已安装 = TRUE
       THEN 执行: wsl --install -d Ubuntu
       ELSE 显示错误并提供解决方案

     STEP 3: 更新 Ubuntu 系统
       IF Ubuntu 已启动 = TRUE
       THEN 执行: sudo apt update && sudo apt upgrade -y

     STEP 4: 安装 nvm 和 Node.js
       执行: curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
       重新加载: source ~/.bashrc
       安装 Node: nvm install --lts

     STEP 5: 安装 AI CLI 工具
       执行: curl -fsSL https://opencode.ai/install | bash
       或: npm install -g opencode-ai

     STEP 6: 安装基础工具
       执行: sudo apt install git python3 build-essential tmux -y

     STEP 7: 配置 Git
       执行: git config --global user.name "用户名"
             git config --global user.email "邮箱"
```

#### 方案 B：Windows 原生

```
IF 用户选择原生方案 AND 操作系统 = Windows
THEN 按顺序执行：

     STEP 1: 安装 Windows Terminal
       IF winget 可用 = TRUE
       THEN 执行: winget install Microsoft.WindowsTerminal
       ELSE 引导下载安装

     STEP 2: 安装 Node.js
       执行: winget install OpenJS.NodeJS.LTS
       或下载安装包

     STEP 3: 安装 Git
       执行: winget install Git.Git

     STEP 4: 安装 Python
       执行: winget install Python.Python.3.11

     STEP 5: 安装 AI CLI 工具
       执行: npm install -g opencode-ai
       或: scoop install extras/opencode
```

### macOS 用户

```
IF 操作系统 = macOS
THEN 按顺序执行：

     STEP 1: 安装 Homebrew
       执行: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

     STEP 2: 安装 Node.js
       执行: brew install node

     STEP 3: 安装 AI CLI 工具
       执行: npm install -g opencode-ai
       或: brew install anomalyco/tap/opencode

     STEP 4: 安装开发工具
       执行: brew install git python tmux

     STEP 5: 配置 Git
       执行: git config --global user.name "用户名"
             git config --global user.email "邮箱"
```

## CLI 全权限配置（可选）

```
IF 用户需要跳过确认直接执行命令
THEN 根据用户偏好的 CLI 工具执行：

     IF CLI工具 = Codex
     THEN 配置: codex --enable web_search_request -m gpt-5.3-codex-max -c model_reasoning_effort="high" --dangerously-bypass-approvals-and-sandbox

     IF CLI工具 = Claude Code
     THEN 配置: claude --dangerously-skip-permissions

     IF CLI工具 = Gemini CLI
     THEN 配置: gemini --yolo
```

---

## Bash 别名配置（可选）

```
IF 用户需要快捷命令
THEN 引导用户在 ~/.bashrc 添加：

     # c - Codex
     alias c='codex --enable web_search_request -m gpt-5.3-codex-max -c model_reasoning_effort="high" --dangerously-bypass-approvals-and-sandbox'

     # cc - Claude Code
     alias cc='claude --dangerously-skip-permissions'

     # g - Gemini CLI
     alias g='gemini --yolo'

     THEN 执行: source ~/.bashrc
```

## 验证检查点

```
验证命令（根据安装的工具）：

  node --version      # Node.js
  npm --version       # npm
  git --version       # Git
  python3 --version    # Python
  opencode --version  # OpenCode CLI

IF 所有命令返回版本号（非错误）
THEN 环境搭建成功
ELSE 报告失败的工具并提供解决方案
```

## 常见问题处理

| 问题 | 诊断条件 | 解决方案 |
|:-----|:---------|:--------|
| WSL2 安装失败 | Hyper-V 相关错误 | 启用 Windows 功能中的 "Linux子系统" |
| nvm 命令未找到 | 终端重启后 | 执行: source ~/.bashrc 或重启终端 |
| npm 安装慢 | 网络问题 | 配置代理或使用淘宝镜像 |
| Git 配置无效 | 每次需要重新配置 | 确保使用了 --global 参数 |
