# 四阶段×十二原则方法论 (4 Phases × 12 Principles)

> **方法论类型**：AI 协作效率框架
> **来源**：12Factor.me
> **AI 执行要点**：提升 AI 协作效率的实践指南

---

## 阶段一：准备 (Preparation)

### 1. 单一真源 (Single Source of Truth)

```
IF 多人协作 THEN
  ├─► 将需求、设计、上下文集中于统一文档
  ├─► 与 AI 协作时直接引用此"真源"
  └─► 禁止信息分散导致上下文混乱
END IF
```

### 2. 提示词先行 (Prompt First)

```
IF 开始任务 THEN
  ├─► 优先编写提示词
  ├─► 明确输入、输出、风格、约束
  └─► 复用经过验证的提示词模板
END IF
```

### 3. 上下文洁净 (Context Hygiene)

```
IF 开始新任务 THEN
  ├─► 开独立会话
  ├─► 避免旧内容干扰
  └─► 定期用一句话总结现状
END IF
```

---

## 阶段二：执行 (Execution)

### 4. 人类在环 (Human-in-the-Loop)

```
IF AI 产出重要结果 THEN
  ├─► 人类负责关键决策与风险把关
  ├─► 对重要功能先进行逻辑验证
  └─► 再合并代码
END IF
```

### 5. 任务块化 (Chunked Work)

```
IF 任务复杂 THEN
  ├─► 拆分为小块（10-30 分钟）
  ├─► 每块结束后立即验证
  └─► 禁止一次性生成 5000 行
END IF
```

### 6. 并行流动 (Parallel Flow)

```
IF AI 工作时 THEN
  ├─► 人类做低切换成本的副任务
  ├─► 准备"副任务清单"
  │     ├─► 文档整理
  │     ├─► 小修复
  │     └─► 代码审查
  └─► 保持节奏不断
END IF
```

---

## 阶段三：协作 (Collaboration)

### 7. 负载预算 (Cognitive Load Budget)

```
IF 全天候黏着 AI 工作 THEN
  └─► 晚上完全耗尽
END IF

建议：
├─► 为 AI 协作设定每日时长上限
└─► 在精神高峰期安排深度审查
```

### 8. 流保护罩 (Flow Protection)

```
IF 高专注流被打断 THEN
  └─► 恢复成本极高
END IF

建议：
├─► 设定专注时段（如 90 分钟）
├─► 屏蔽通知与打扰
└─► AI 交互也在专注流中批量进行
```

### 9. 可复现性 (Reproducible Sessions)

```
IF 需要复盘或排错 THEN
  ├─► 保存 Prompt
  ├─► 保存 AI 版本
  ├─► 保存变更原因
  └─► 保存到代码库或知识库
END IF
```

---

## 阶段四：迭代 (Iteration)

### 10. 休息反思 (Rest & Reflection)

```
IF 冲刺结束 THEN
  ├─► 花 5 分钟复盘 AI 产出与预期差异
  ├─► 更新 Prompt 模板
  └─► 积累"踩坑记录"
END IF
```

### 11. 技能均衡 (Skill Parity)

```
IF 完全依赖 AI THEN
  └─► 失去手写能力与技术洞察力
END IF

建议：
├─► 持续学习领域知识
├─► 持续学习代码审查技巧
└─► 对 AI 输出保持独立判断
```

### 12. 好奇文化 (Culture of Curiosity)

```
IF 面对 AI 答案 THEN
  ├─► 先问"为什么"
  ├─► 再问"还能更好吗"
  └─► 团队分享改进思路
END IF

IF 对 AI 方案照单全收 THEN
  └─► 从不质疑（危险）
END IF
```
