# Vibe Coding 问题-解决方案映射 (Problem-Solution Map)

> **文档类型**：问题 → 原因 → 解决方案 → 规则引用
> **用途**：快速定位常见问题的解决方案
> **格式**：问题描述 | 根因分析 | 解决方案 | 关联规则

---

## 一、AI 对话问题

### 1.1 AI 生成的代码跑不起来

| 属性 | 内容 |
|:---|:---|
| **问题** | AI 生成的代码执行失败 |
| **根因** | 上下文不足、AI 幻觉、知识过时 |
| **解决方案** | 提供完整错误信息、运行环境、相关代码上下文 |
| **预防规则** | 提供官方文档链接作为参考 |
| **关联协议** | `core-protocols.md` - 上下文管理规则 |

```
触发词："代码跑不起来"、"报错"、"不工作"
解决路径：
1. 复制完整错误信息
2. 说明运行环境（Python 版本、操作系统等）
3. 提供相关代码片段
4. 如有必要，提供最小复现示例
```

### 1.2 AI 反复修改同一问题

| 属性 | 内容 |
|:---|:---|
| **问题** | AI 陷入修改循环，无法解决问题 |
| **根因** | 思路错误、上下文混乱、问题定义不清 |
| **解决方案** | 换个思路描述、开新对话、简化问题范围 |
| **关联协议** | `decision-trees.md` - handle_loop() |

```
触发词："一直修不好"、"陷入循环"、"改了好几遍"
解决路径：
1. 换个角度描述问题
2. 提供更多上下文信息
3. 简化问题范围，聚焦核心
4. 如仍无效，开新对话重试
```

### 1.3 AI 幻觉（编造不存在的 API）

| 属性 | 内容 |
|:---|:---|
| **问题** | AI 生成不存在的 API、函数或参数 |
| **根因** | 模型知识过时、训练数据不完整 |
| **解决方案** | 提供官方文档链接、直接复用成熟库 |
| **预防规则** | `core-protocols.md` - 零幻觉规则 |
| **胶水编程** | `fundamentals/glue-coding.md` |

```
触发词："这个 API 是这样的"、"应该有这个函数"
解决路径：
1. 验证 API 是否存在（查官方文档）
2. IF 不存在 → 使用胶水编程原则，搜索成熟替代
3. 提供官方文档给 AI 作为参考
4. 要求 AI 标注不确定的部分
```

### 1.4 AI 忘记之前的对话

| 属性 | 内容 |
|:---|
| **问题** | AI 在新对话或长对话后忘记之前约定 |
| **根因** | 上下文窗口限制、Token 耗尽 |
| **解决方案** | 使用 memory-bank、定期总结关键信息 |
| **关联文件** | memory-bank/@progress.md, @architecture.md |

```
触发词："你之前说的"、"我们刚才确定"
解决路径：
1. 重新提供关键上下文
2. 更新 memory-bank 持久化
3. 使用 /clear 或 /compact 清理上下文
4. 新对话开始时先读取 memory-bank
```

### 1.5 AI 不理解我的需求

| 属性 | 内容 |
|:---|:---|
| **问题** | AI 理解与人类意图不符 |
| **根因** | 描述模糊、歧义、未提供示例 |
| **解决方案** | 现象学还原、提供输入输出示例 |
| **关联方法** | `concept-index.md` - 现象学还原 |

```
触发词："不对"、"不是这个意思"、"理解错了"
解决路径：
1. 执行现象学还原：现象/意图/情境/边界
2. 提供具体输入输出示例
3. 用一句话明确核心目标
4. 逐步描述而非一次性描述
```

## 二、环境配置问题

### 2.1 死活配不好环境

| 属性 | 内容 |
|:---|:---|
| **问题** | Python/Node 环境配置失败 |
| **根因** | 全局污染、版本冲突、权限问题 |
| **解决方案** | 删除重来，使用虚拟环境隔离 |
| **关联文档** | `fundamentals/common-pitfalls.md` - 虚拟环境 |

```
触发词："环境配不好"、"装不上包"、"找不到 python"
解决路径：
1. rm -rf .venv  # 删除旧环境
2. python -m venv .venv  # 重新创建
3. source .venv/bin/activate  # 激活
4. pip install -r requirements.txt  # 重新安装
```

### 2.2 命令找不到

| 属性 | 内容 |
|:---|:---|
| **问题** | 系统命令或工具找不到 |
| **根因** | 环境变量未配置、未激活虚拟环境 |
| **解决方案** | 检查 PATH、激活虚拟环境、重启终端 |

```
触发词："command not found"、"找不到"
解决路径：
1. 检查是否在正确的虚拟环境中
2. 检查 PATH 环境变量
3. 确认工具已正确安装
4. 重启终端或重新登录
```

### 2.3 网络问题

| 属性 | 内容 |
|:---|:---|
| **问题** | GitHub 访问慢、API 调用失败、下载超时 |
| **根因** | 网络限制、代理配置不全、源在国外 |
| **解决方案** | 配置代理、换国内镜像、使用 SSH |
| **关联文档** | `fundamentals/common-pitfalls.md` - 网络配置 |

```
触发词："timeout"、"访问不了"、"连接失败"
解决路径：
1. 检查代理配置：export http_proxy/https_proxy
2. Git 配置代理：git config --global http.proxy
3. 换国内镜像：pip install -i https://pypi.tuna.tsinghua.edu.cn/simple
4. 使用 SSH 替代 HTTPS clone
```

## 三、代码问题

### 3.1 代码文件太大

| 属性 | 内容 |
|:---|:---|
| **问题** | 文件太大超出上下文限制，AI 处理困难 |
| **根因** | 未模块化、职责混杂 |
| **解决方案** | 拆分文件、只给 AI 相关部分 |
| **关联规则** | `core-protocols.md` - 模块化规则 |

```
触发词："太长了"、"上下文不够"、"处理不了"
解决路径：
1. 按功能拆分文件
2. 每个文件职责单一
3. 只提供 AI 当前任务相关的文件
4. 必要时创建摘要文件
```

### 3.2 代码越改越乱

| 属性 | 内容 |
|:---|:---|
| **问题** | 代码质量下降，越来越难维护 |
| **根因** | 缺乏规划、职责不清、重复代码 |
| **解决方案** | 重构、模块化、遵循 SOLID/DRY |
| **关联原则** | `concept-index.md` - 单一职责、DRY |

```
触发词："乱了"、"不好维护"、"改一处坏多处"
解决路径：
1. 重新阅读 architecture.md 理解架构
2. 识别职责混淆的模块
3. 拆分职责，重构代码
4. 编写测试确保重构正确
```

### 3.3 改了代码没生效

| 属性 | 内容 |
|:---|:---|
| **问题** | 修改代码后运行结果不变 |
| **根因** | 缓存问题、未保存、路径错误、运行错误文件 |
| **解决方案** | 清缓存、确认保存、检查路径、重启服务 |

```
触发词："改了没用"、"没变化"、"还是旧的"
解决路径：
1. 确认文件已保存
2. 清缓存（浏览器/服务器）
3. 检查是否运行了正确的文件
4. 重启服务
5. 检查是否有其他同名文件覆盖
```

### 3.4 合并代码冲突

| 属性 | 内容 |
|:---|:---|
| **问题** | Git 合并时出现冲突无法解决 |
| **根因** | 同时修改同一文件、协作问题 |
| **解决方案** | 让 AI 帮忙解决、提供冲突内容 |

```
触发词："conflict"、"冲突"
解决路径：
1. 查看冲突文件内容
2. 理解冲突原因
3. 决定保留哪些修改
4. 手动解决或让 AI 帮助解决
5. git add 标记解决
```

## 四、测试问题

### 4.1 不知道测什么

| 属性 | 内容 |
|:---|:---|
| **问题** | 不知道代码需要测试什么 |
| **根因** | 缺乏测试思维、边界不清 |
| **解决方案** | 测试边界条件、异常情况、核心逻辑 |
| **关联方法** | `decision-trees.md` - 设计测试策略 |

```
触发词："要测什么"、"不知道怎么写测试"
解决路径：
1. 列出函数的输入类型和范围
2. 识别边界条件（空值、最大值、特殊字符）
3. 识别异常情况（网络失败、超时、权限）
4. 识别核心业务逻辑路径
5. 为每个场景编写测试
```

### 4.2 测试太慢

| 属性 | 内容 |
|:---|:---|
| **问题** | 测试执行时间过长影响开发效率 |
| **根因** | 测试粒度太大、依赖外部服务、数据库操作 |
| **解决方案** | 单元测试为主、Mock 外部依赖、优化测试数据 |

```
触发词："测试太慢"、"跑不动"
解决路径：
1. 分离慢速测试和快速测试
2. Mock 外部依赖（数据库、API）
3. 使用内存数据库替代真实数据库
4. 优化测试数据规模
5. 并行执行独立测试
```

## 五、性能问题

### 5.1 API 响应慢

| 属性 | 内容 |
|:---|:---|
| **问题** | 接口响应时间过长 |
| **根因** | 数据库查询慢、未使用索引、未缓存 |
| **解决方案** | 加索引、缓存、分页、N+1 查询优化 |
| **关联文档** | `fundamentals/language-layers.md` - L10 资源模型 |

```
触发词："响应慢"、"超时"
解决路径：
1. 添加日志定位慢查询
2. 检查是否有 N+1 查询问题
3. 添加数据库索引
4. 实现缓存机制
5. 分页处理大数据量
```

### 5.2 内存泄漏

| 属性 | 内容 |
|:---|:---|
| **问题** | 程序运行时间越长内存占用越高 |
| **根因** | 未释放资源、事件监听未移除、循环引用 |
| **解决方案** | 检查资源清理、移除监听器、使用弱引用 |
| **关联概念** | `fundamentals/language-layers.md` - L5 错误模型 |

```
触发词："内存越来越高"、"越来越卡"
解决路径：
1. 使用 profiler 定位内存增长点
2. 检查是否有未关闭的文件/连接
3. 检查是否有未移除的事件监听器
4. 检查是否有循环引用
5. 使用弱引用（weakref）替代强引用
```

## 六、安全问题

### 6.1 API Key 泄露

| 属性 | 内容 |
|:---|:---|
| **问题** | 密钥/Token 被提交到 Git |
| **根因** | 缺乏安全意识、未使用 .gitignore |
| **解决方案** | 从 Git 历史删除、更换密钥、使用环境变量 |
| **关联规则** | `core-protocols.md` - 凭证管理规则 |

```
触发词："不小心提交了"、"key 泄露了"
解决路径：
1. 立即更换泄露的密钥
2. 从 Git 历史中删除：git filter-branch 或 BFG Repo-Cleaner
3. 将密钥添加到 .gitignore
4. 以后使用环境变量读取
```

### 6.2 SQL 注入

| 属性 | 内容 |
|:---|:---|
| **问题** | 用户输入未转义导致 SQL 注入 |
| **根因** | 字符串拼接 SQL、使用用户输入 |
| **解决方案** | 使用参数化查询、使用 ORM |
| **关联规则** | `core-protocols.md` - 输入验证规则 |

```
触发词："SQL 注入"、"数据安全问题"
解决路径：
1. 不使用字符串拼接 SQL
2. 使用参数化查询
3. 使用 ORM 框架
4. 对用户输入进行验证和转义
```

## 七、部署问题

### 7.1 本地能跑，部署失败

| 属性 | 内容 |
|:---|:---|
| **问题** | 本地开发正常，部署环境报错 |
| **根因** | 环境差异、版本不一致、环境变量缺失 |
| **解决方案** | 检查环境差异、一致性配置 |
| **关联文档** | `fundamentals/common-pitfalls.md` - 部署相关 |

```
触发词："本地正常"、"部署报错"
解决路径：
1. 检查 Node/Python 版本是否一致
2. 检查环境变量是否配置
3. 检查依赖版本是否一致（requirements.txt）
4. 检查构建命令和环境
5. 查看部署日志定位问题
```

### 7.2 CORS 跨域错误

| 属性 | 内容 |
|:---|:---|
| **问题** | 浏览器报 CORS 错误 |
| **根因** | 后端未配置 CORS、域名不匹配 |
| **解决方案** | 后端添加 CORS 中间件、配置允许的域名 |

```
触发词："CORS"、"跨域"
解决路径：
1. 后端添加 CORS 中间件
2. 配置允许的来源域名
3. 配置允许的方法和头
4. 如需要，配置凭证（credentials）
```

## 八、大模型使用问题

| 属性 | 内容 |
|:---|:---|
| **问题** | 输入内容超出模型 Token 限制 |
| **根因** | 上下文太长、文件太大 |
| **解决方案** | 精简上下文、拆分文件、使用 /clear |
| **关联规则** | `core-protocols.md` - 上下文管理规则 |


| 属性 | 内容 |
