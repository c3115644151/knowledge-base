## 常用提示词模板

### 1. 基础编程提示词

```
你是一个专业的 AI 编程助手。我想用 Vibe Coding 的方式开发一个项目。

请先问我：
1. 你想做什么项目？（一句话描述）
2. 你熟悉什么编程语言？（不熟悉也没关系）
3. 你的操作系统是什么？

然后帮我：
1. 推荐最简单的技术栈
2. 生成项目结构
3. 一步步指导我完成开发

要求：每完成一步问我是否成功，再继续下一步。
```

### 2. 现象学还原提示词

```
先做现象学还原：
不要推测原因。
输出格式：
1. 现象：[具体描述实际观察到的情况]
2. 意图：[具体描述期望的行为]
3. 情境：[环境约束信息]
4. 边界：[明确排除的范围]
5. 未确定项：[需要补充的信息]
6. MRE（最小可复现示例）

然后再给最小修复与测试。
```

### 3. 正反合迭代提示词

```
按正反合输出：
1) 最小可运行实现（正）
2) 反例与失败模式+测试（反）
3) 综合后的重构方案与最终代码（合）
```

### 4. 代码审查提示词

```
# 角色
你是一名世界级系统架构师 + 质量工程专家。

# 任务
针对我的项目需求，构建一套"可执行、可审计、可复用"的完整检查清单。

# 输入
- 项目目标：
- 技术栈/运行环境：
- 关键约束：
- 需求规范集合：

# 输出要求
1. 需求语义解构
2. 检查点枚举（每条可判定 Yes/No/Unknown）
3. 逐项逻辑检查
4. 规范间冲突分析
```
