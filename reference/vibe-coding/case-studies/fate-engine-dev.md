# Fate Engine 命理引擎项目

## 项目背景

Fate Engine 是一个命理分析引擎项目，专注于八字、K 线等数据的分析与处理。涉及复杂的算法实现和数据可视化需求。

## 核心技术栈

- **语言**: Python
- **依赖模式**: 强依赖复用（直连成熟仓库源码）
- **可视化**: ASCII 字符绘制 K 线图

## 关键实现要点

### IF-THEN 规则

1. **IF** 需要复用外部成熟仓库的能力（如八字算法、数据模块）
   **THEN** 使用 `sys.path` 添加本地路径，直接导入完整模块，禁止子集封装或逻辑重写

2. **IF** 需要生成 K 线图可视化
   **THEN** 使用 ASCII 字符绘制，保持终端兼容性

3. **IF** 需要确保代码完整性
   **THEN** 执行系统性完整性检查，验证所有依赖模块真实参与执行

## 胶水开发规范

### 核心原则

> 尽可能减少自行实现的底层与通用逻辑，优先、直接、完整地复用既有成熟仓库与库代码。

### 依赖结构要求

```python
sys.path.append('/path/to/libs/external/github/*')
from datas import *        # 完整数据模块
from sizi import summarys  # 完整算法实现
```

### 明确禁止

- Mock / Stub 替代
- Demo / 示例代码替代
- 功能裁剪或逻辑重写
- 复制代码到当前项目后修改

### 项目职责边界

**允许**：
- 业务流程编排（Orchestration）
- 模块组合与调度
- 参数配置与调用组织
- 输入输出适配

**禁止**：
- 重复实现算法
- 重写已有数据结构
- 将复杂逻辑从依赖库中"拆出来自己写"

## 可复用经验总结

1. **强依赖复用模式**：评价标准不是"写了多少代码"，而是"是否正确站在成熟系统之上"
2. **问题描述模板化**：确保任何技术人员或 LLM 都能无需追问理解问题全貌
3. **完整性检查流程**：代码生成后必须验证所有导入模块真实参与执行
