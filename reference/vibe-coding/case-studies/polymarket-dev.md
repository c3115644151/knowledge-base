# Polymarket 套利项目

## 项目背景

Polymarket 是一个预测市场平台，用户可以在上面交易事件结果（如"某事件是否发生"）。核心交易资产为 YES 和 NO 股份，猜对一方价值 $1，猜错归零。理论公式为：**YES 价格 + NO 价格 = $1**。

实际交易中，由于情绪化下单、全球时差、低流动性砸盘、跨平台价差等因素，价格经常偏离理论值，产生套利机会。

## 核心技术栈

- **语言**: Python
- **数据源**: Polymarket API、Kalshi API
- **可视化**: ASCII 字符绘制 K 线图
- **工具**: 监控机器人程序化交易

## 关键实现要点

### IF-THEN 规则

1. **IF** 市场突发消息导致 YES 价格飙升（0.60）而 NO 价格反应滞后（0.35）
   **THEN** 买入 1 股 YES + 1 股 NO（总价 $0.95），结算时必得 $1.00，锁定 5.2% 利润

2. **IF** 美国凌晨发布新闻，美国交易员买爆 YES（0.70），亚洲 NO 未同步（0.33）
   **THEN** 反向操作卖出 1 股 YES + 1 股 NO（收入 $1.03），结算归还 $1.00，锁定 $0.03

3. **IF** 低流动性事件中大额卖单砸盘 YES（0.45）而 NO 未反应（0.50）
   **THEN** 监控机器人程序化买入被低估资产组合，等待恢复或持有至结算

4. **IF** Polymarket 和 Kalshi 同事件价格出现差异（Polymarket NO $0.20，Kalshi YES $0.75）
   **THEN** 跨平台同时买入 NO（$0.20）+ YES（$0.75）= $0.95，结算得 $1.00

## 遇到的问题与解决方案

| 问题 | 解决方案 |
|:---|:---|
| 价格错位窗口期短 | 自动化监控机器人实时捕捉 |
| 跨平台 API 整合 | 统一的数据采集和解析模块 |
| ASCII 可视化展示 | 使用 prompt 生成 K 线图 ASCII 字符画 |

## 可复用经验总结

1. **胶水代码开发模式**：遵循"强依赖复用"原则，优先直接复用成熟仓库，仅编写最小业务层
2. **完整性检查**：代码生成后进行系统性检查确保功能完整
3. **ASCII 可视化**：适合终端环境下的数据展示，便于快速理解市场状态
