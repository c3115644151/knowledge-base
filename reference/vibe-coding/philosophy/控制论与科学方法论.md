# 控制论与科学方法论

## 核心概念定义

### 可能性空间
- **定义**：问题或任务存在多种可能的解决路径，呈现为一个可能性空间
- **AI视角**：生成代码时，每个决策点都分支出多种可能性
- **量化指标**：M/m = 控制能力（初始可能性/当前可能性）

### 负反馈循环
- **定义**：通过不断比较当前状态与目标差距，迭代修正的机制
- **作用**：将有限的单次能力累积放大，实现超越单次上限的精确控制
- **收敛条件**：差距逐步缩小直至达到可接受阈值

### 正反馈机制
- **定义**：输出放大输入的自我增强循环
- **风险**：可能导致恶性循环或过早收敛
- **应用**：在需要快速探索时加速发散，在收敛时必须抑制

### 信息与不确定性
- **定义**：信息 = 可能性空间的收缩程度（比特 = log₂(M/m)）
- **本质**：每条反馈都是对不确定性的消除
- **原则**：控制能力受限于可获得的信息量

### 黑箱模型
- **定义**：对未知系统通过输入-输出观察构建认知模型
- **迭代**："实践-理论-实践"的负反馈循环
- **收敛条件**：
  1. 理论具有可证伪性（提供清晰信息）
  2. 反馈速度 > 系统变化速度
  3. 调节幅度适度
  4. 结果与理论存在可靠判别关系

## 触发场景

| 场景 | 判断标准 | 方法论选择 |
|:-----|:---------|:-----------|
| 试错无进展 | 多次尝试后可能性空间未缩小 | 引入负反馈机制 |
| 方案发散过多 | 生成结果偏离核心目标 | 建立约束缩小可能性空间 |
| 收敛遇瓶颈 | 同一问题反复无法解决 | 重构黑箱模型/更换探索维度 |
| 快速验证 | 需要在多条路径中快速筛选 | 正反馈加速探索后引入负反馈收敛 |
| 复杂系统分析 | 涉及多层次因果关系 | 建立相对孤立系统模型 |

## 执行协议

### Protocol 1: 负反馈收敛循环
```
1. 定义目标状态（清晰的完成标准）
2. 记录当前状态（每次迭代的快照）
3. 计算差距（目标 - 当前）
4. 施加修正（针对差距最大的维度）
5. 验证修正效果（可观测的输出变化）
6. 重复直到收敛或触发终止条件
```

### Protocol 2: 可能性空间收缩
```
1. 识别所有可能的解决路径（列出分支）
2. 评估每条路径的可行性（信息量/风险/资源）
3. 剪枝低可能性路径（排除明显不可行）
4. 聚焦最高可能性路径（扩大控制能力）
5. 如遇阻滞，回到步骤1重新评估
```

### Protocol 3: 黑箱测试与模型修正
```
1. 对未知系统施加可控输入
2. 观察可观测输出
3. 记录输入-输出映射
4. 构建/更新内部模型
5. 预测新输入的输出
6. 用实践验证预测
7. 根据差异修正模型
```

### Protocol 4: 稳态与质变判断
```
1. 检测当前状态是否稳定（输出是否收敛）
2. 如稳定：持续优化，警惕微小涨落
3. 如不稳定：判断是渐变还是飞跃
   - 渐变条件：存在稳定中间态 → 逐步推进
   - 飞跃条件：稳定态消失 → 果断跃迁，接受"矫枉过正"
```

### Protocol 5: 反馈质量保障
```
检查清单：
□ 反馈是否提供可证伪的预测？
□ 反馈速度是否追得上变化？
□ 调节幅度是否适中（不过度）？
□ 结果能否清晰判别理论真伪？

如有任一项不满足，暂停当前循环，重构问题框架
```

## AI行为决策指南

### 决策树
```
当前探索是否有效？
├── 否 → 评估反馈机制是否建立
│         ├── 无反馈 → 引入负反馈循环
│         └── 反馈过慢 → 加速验证频率
└── 是 → 可能性空间是否收敛？
          ├── 否 → 检查是否需要"过正"修正
          └── 是 → 验证收敛方向是否正确
                   ├── 正确 → 继续收敛
                   └── 偏离 → 重构黑箱模型
```

### 关键原则
1. **控制即选择**：每个决策都是对可能性空间的选择
2. **信息即控制**：获取信息是缩小可能性的唯一途径
3. **收敛靠反馈**：无反馈则无真正的控制
4. **可证伪性**：理论必须能够被证明错误才有价值
5. **稳态检测**：持续监测系统是否处于稳定状态

## 收敛判断标准

| 条件 | 标准 |
|:-----|:-----|
| 功能收敛 | 核心需求得到满足，边界情况可接受 |
| 质量收敛 | 测试通过率 > 95%，无高危警告 |
| 复杂度收敛 | 代码结构清晰，依赖关系稳定 |
| 信息收敛 | 所有关键不确定性已消除 |
